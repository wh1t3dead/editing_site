# Wh1te Dead Portfolio

Статический сайт-портфолио под GitHub Pages.

## Что внутри

- `index.html` - основная страница портфолио.
- `admin.html` - локальная админка для редактирования тем и видео.
- `content.js` - весь контент сайта в одном файле.
- `app.js` - рендер главной страницы.
- `admin.js` - логика редактора и экспорта `content.js`.
- `styles.css` - все стили.

## Как обновлять сайт

1. Открой `admin.html`.
2. Добавь или измени темы и видео.
3. Нажми `Скачать content.js`.
4. Замени этим файлом существующий `content.js` в репозитории.
5. Закоммить и запушь изменения на GitHub.

## Как загрузить на GitHub Pages

1. Создай новый репозиторий на GitHub.
2. Загрузи в него все файлы из этой папки.
3. В настройках репозитория открой `Settings -> Pages`.
4. Включи публикацию из ветки `main` и корня репозитория (`/root`).
5. GitHub выдаст ссылку на сайт.

## Важно про админку

GitHub Pages - это статический хостинг. Поэтому `admin.html` помогает редактировать контент, но не может сама сохранять изменения в GitHub без отдельного внешнего сервиса или OAuth-настройки.
